Unpacking ASAR files

ASAR (pronounced "a-sar") stands for "Atom Shell Archive." It's a packaging format used in Electron applications to bundle together various files, including JavaScript, HTML, CSS, and other assets, into a single archive. This archive format is similar to a ZIP file but optimized for Electron apps. ASAR files help streamline the distribution and loading of application resources, improving performance and efficiency. Electron apps often rely on these ASAR archives to package and organize their code and assets.

For security researchers examining Electron apps for vulnerabilities, unpacking ASAR files is an essential step. The reason is that ASAR archives can obscure the underlying code and assets, making it challenging to analyze and identify security issues effectively. By unpacking ASAR files, researchers gain direct access to the application's source code, allowing them to inspect it thoroughly, identify potential security flaws, and assess the app's security posture. This level of access is crucial for understanding how an Electron app functions and for conducting comprehensive security audits, penetration testing, and vulnerability assessments. Without unpacking ASAR files, researchers would be limited in their ability to assess and uncover potential vulnerabilities, hindering their efforts to enhance the security of Electron applications.

If you have NPM installed, then unpacking an ASAR file is very easy and can be accomplished with the following commands:

npx asar extract app.asar destfolder

npx asar extract-file app.asar main.js

Once you have extracted the contents of the ASAR file, then you can start reviewing it for information like endpoints or functions that you would usually not know about. Keep in mind that javascript files will often be obfuscated, so tools like https://deobfuscate.io/ can be very helpful when it comes to making the formating readable again.
